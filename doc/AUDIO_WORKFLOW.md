# 播客音频加工流程产品设计手册

> 版本: 1.0 | 更新日期: 2026-02-27

---

## 一、概述

本文档详细描述 Podcast Studio 中**音频加工**的核心工作流程，包括：

1. **逐字稿生成** - 将新闻内容转换为播客脚本
2. **人工精调** - 用户编辑优化逐字稿
3. **语音合成** - 将逐字稿转为语音文件
4. **音频合并** - 整合开场、新闻、结尾，生成最终节目

## 三、详细流程设计

### 3.1 逐字稿生成 (Script Generation)

#### 流程说明

```
用户点击"生成脚本" → 调用 LLM API → 生成播客逐字稿 → 保存到数据库
```

#### 触发条件

- 用户在新闻列表中点击单条新闻的「生成脚本」按钮
- 用户批量选择多条新闻点击「批量生成脚本」
- 自动模式下，系统根据预设模板自动触发

#### 数据模型

```typescript
interface EpisodeNews {
  id: number
  episode_id: number
  news_id: number
  order: number
  
  // 逐字稿相关
  status: 'pending' | 'generating' | 'script_done' | 'audio_done' | 'error'
  prompt: string          // 自定义提示词
  original_content: string  // 发送给 LLM 的原文
  script: string        LM 生成的逐字稿
 // L  script_version: number // 逐字稿版本号
  
  // 音频相关
  audio_url: string      // 生成的音频文件 URL
  audio_duration: number // 音频时长(秒)
  
  // 元数据
  error_message: string | null
  created_at: datetime
  updated_at: datetime
}
```

#### API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/episodes/{id}/news/{newsId}/generate-script` | 生成单条逐字稿 |
| POST | `/episodes/{id}/generate-scripts` | 批量生成逐字稿 |
| PUT | `/episodes/{id}/news/{newsId}/script` | 更新逐字稿 |

#### 提示词模板

```yaml
script_generation:
  default: |
    你是一个专业的播客主持人。请将以下新闻内容改写成播客逐字稿。
    
    要求：
    1. 语言口语化，适合朗读
    2. 保持新闻核心信息
    3. 适当添加过渡语
    4. 长度控制在 500-1000 字
    5. 格式：男：... 女：... 或 主持人：...
    
    新闻内容：
    {content}

  tech: |
    你是一个科技播客主持人。请用专业但易懂的语言解读以下科技新闻。
    ...
```

### 3.2 人工精调 (Manual Refinement)

#### 流程说明

```
查看原始内容 → 编辑逐字稿 → 保存修改 → (可选)重新生成语音
```

#### 核心功能

| 功能 | 说明 |
|------|------|
| **原文对照** | 左侧显示发送给 LLM 的原文，右侧显示生成的逐字稿 |
| **-inline 编辑** | 直接在文本框内修改，无需弹窗 |
| **版本管理** | 每次保存生成新版本，支持版本对比和回退 |
| **预览朗读** | 点击按钮语音朗读当前逐字稿 (TTS) |
| **重置** | 一键恢复为 LLM 生成的原始版本 |

#### UI 设计


## 四、状态管理

### 4.1 新闻处理状态

| 状态 | 值 | 说明 | UI 颜色 |
|------|-----|------|---------|
| pending | 待处理 | 未开始生成 | 灰色 |
| generating | 生成中 | 正在调用 LLM/TTS | 蓝色 (动画) |
| script_done | 脚本完成 | 逐字稿已生成 | 绿色 |
| audio_done | 音频完成 | 语音已生成 | 橙色 |
| error | 错误 | 生成失败 | 红色 |

## 五、错误处理

### 5.1 错误类型

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| LLM_ERROR | LLM 调用失败 | 显示错误信息，提供重试按钮 |
| TTS_ERROR | TTS 调用失败 | 显示错误信息，提供重试 |
| AUDIO_MERGE_ERROR | 音频合并失败 | 提示具体片段问题 |
| NETWORK_ERROR | 网络问题 | 自动重试 3 次 |
| QUOTA_EXCEEDED | API 配额用尽 | 提示用户联系管理员 |

## 七、API 汇总

### 逐字稿相关

| 方法 | 路径 | Body | Response |
|------|------|------|----------|
| POST | `/episodes/{id}/news/{newsId}/generate-script` | `{prompt?: string}` | `{script: string, version: number}` |
| PUT | `/episodes/{id}/news/{newsId}/script` | `{script: string}` | EpisodeNews |
| GET | `/episodes/{id}/news/{newsId}/versions` | - | ScriptVersion[] |

### 音频相关

| 方法 | 路径 | Body | Response |
|------|------|------|----------|

| 方法 | 路径 | Body | Response |

## 八、总结

本手册定义了 Podcast Studio 音频加工的完整产品流程，涵盖：

1. **逐字稿生成** - AI 自动将新闻转为播客脚本
2. **人工精调** - 支持-inline 编辑和版本管理
3. **语音合成** - 多种音色选择，支持批量生成
4. **音频合并** - 时间线视图，开场/结尾/BGM 配置

后续开发请以此文档为准。
